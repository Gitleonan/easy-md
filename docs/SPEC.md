# md++ (easy-md) — 功能规格清单

> **应用显示名:** md++
> **GitHub 项目名:** easy-md
> **定位:** Windows 平台轻量、便捷的 Markdown 纯阅读器；架构预留 macOS 适配能力。

---

## 1. 核心定位

- **纯阅读器**: 不提供编辑能力，专注内容呈现。
- **便捷**: 多种入口快速打开文件（右键关联 / 拖拽 / 对话框 / 最近文件）。
- **跨端预留**: 基于 Tauri v2，前端代码与平台无关，后端 Rust 仅做平台隔离。

---

## 2. 功能清单

### 2.1 Markdown 渲染

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| R-01 | 基础语法 | 标题、段落、列表、引用、链接、图片、表格、分割线、行内代码 | P0 |
| R-02 | 代码高亮 | 基于 `shiki`，支持主流语言与 VS Code 主题 | P0 |
| R-03 | 数学公式 | 基于 `@traptitech/markdown-it-katex` 支持 `$...$` / `$$...$$` | P0 |
| R-04 | Mermaid 图表 | 流程图、时序图、甘特图、饼图等 | P0 |
| R-05 | 任务列表 | `- [x]` / `- [ ]` checkbox 列表（只读，不可勾选） | P1 |
| R-06 | 脚注 | `[^1]` 引用与底部脚注定义 | P1 |
| R-07 | 高亮标记 | `==高亮==` 渲染为黄色背景标记 | P1 |
| R-08 | 自动链接 | 裸链接自动转为可点击超链接 | P1 |

### 2.2 文件打开

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| F-01 | 右键菜单关联 | Windows 右键 `.md` 文件 → "用 md++ 打开"；安装时注册 | P0 |
| F-02 | 拖拽打开 | 拖拽文件到窗口直接打开为新标签 | P0 |
| F-03 | 文件对话框 | 应用内"打开文件"按钮，调用系统文件选择器 | P0 |
| F-04 | 最近文件 | 启动欢迎页列出最近打开文件（持久化到本地） | P1 |
| F-05 | 系统关联启动 | 双击关联的 `.md` 通过命令行参数传入应用 | P0 |
| F-06 | 多文件参数 | 命令行传入多个文件路径，启动时全部打开为标签 | P2 |

### 2.3 多标签页

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| T-01 | 顶部标签栏 | 浏览器风格顶部标签，显示文件名 | P0 |
| T-02 | 新建/关闭 | 标签可关闭；关闭最后一个标签显示欢迎页 | P0 |
| T-03 | 切换记忆 | 各标签独立保留滚动位置、TOC 展开状态 | P1 |
| T-04 | 拖拽排序 | 标签可拖拽调整顺序 | P2 |
| T-05 | 未修改自动刷新 | 文件被外部修改时标签内自动刷新（无未保存概念，直接刷新） | P0 |

### 2.4 阅读界面

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| U-01 | 双栏布局 | 左侧 TOC 目录 + 右侧内容区 | P0 |
| U-02 | TOC 自动生成 | 从标题（h1-h6）自动生成目录树，点击跳转 | P0 |
| U-03 | 滚动联动 | 内容滚动时高亮当前所在 TOC 项 | P1 |
| U-04 | TOC 折叠 | 支持折叠/展开 TOC 面板 | P1 |
| U-05 | 内容宽度限制 | `max-width` 居中（约 800px），长文阅读最佳 | P0 |
| U-06 | 主题切换 | 浅色 / 深色 / 跟随系统，切换即时生效 | P0 |
| U-07 | 简洁极简风 | 类 Typora/GitHub 排版，UI 不喧宾夺主 | P0 |

### 2.5 图片处理

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| I-01 | 相对路径 | 以 MD 文件所在目录为基准解析相对路径 | P0 |
| I-02 | 绝对路径 | 支持本地绝对路径图片 | P0 |
| I-03 | 网络图片 | 支持 `http(s)://` 网络图片 | P0 |
| I-04 | 加载失败占位 | 图片不存在时显示友好占位符 | P1 |
| I-05 | 点击放大 (Lightbox) | 点击图片弹出 lightbox，支持缩放/拖动查看 | P0 |

### 2.6 搜索

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| S-01 | 全文搜索 | 在当前打开文件内搜索关键词，高亮所有匹配 | P0 |
| S-02 | 搜索导航 | 上一个/下一个匹配跳转，显示"第 N/总 M 个" | P0 |
| S-03 | 快捷键唤起 | `Ctrl+F` 唤起搜索栏 | P0 |

### 2.7 导出

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| E-01 | 导出 PDF | 渲染后内容导出为 PDF | P0 |
| E-02 | 导出 HTML | 导出为带样式的独立 HTML 文件 | P1 |
| E-03 | 系统打印 | 调用系统打印对话框（含"打印到 PDF"） | P1 |
| E-04 | 复制渲染内容 | 一键复制渲染后富文本，可粘贴到 Word/邮件 | P0 |

### 2.8 文件监听

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| W-01 | 外部修改刷新 | 文件被外部编辑后自动重新加载渲染 | P0 |
| W-02 | 防抖处理 | 避免频繁保存导致的连续刷新（防抖 300ms） | P1 |

### 2.9 快捷键

| 编号 | 功能 | 快捷键 | 优先级 |
|------|------|--------|--------|
| K-01 | 打开文件 | `Ctrl+O` | P0 |
| K-02 | 关闭标签 | `Ctrl+W` | P0 |
| K-03 | 切换主题 | `Ctrl+Shift+T` | P1 |
| K-04 | 全文搜索 | `Ctrl+F` | P0 |
| K-05 | 导出 PDF | `Ctrl+P` | P1 |
| K-06 | 复制渲染内容 | `Ctrl+Shift+C` | P1 |

### 2.10 跨端适配预留

| 编号 | 功能 | 说明 | 优先级 |
|------|------|------|--------|
| X-01 | 平台隔离层 | 后端 Rust 命令按平台隔离（`#[cfg(windows)]` / `#[cfg(target_os = "macos")]`） | P0 |
| X-02 | 快捷键平台化 | 快捷键定义按平台区分（Mac 用 `Cmd`） | P1 |
| X-03 | 无平台耦合代码 | 前端不出现 `process.platform` 等平台判断硬编码 | P0 |

## 3. 非功能需求

| 维度 | 要求 |
|------|------|
| **启动速度** | 冷启动 < 1s（WebView 已就绪后） |
| **包体积** | 安装包 < 15MB（Tauri Rust 内核 + 资源） |
| **内存占用** | 空载 < 100MB，单文件渲染 < 200MB |
| **支持平台** | 首发 Windows 10/11（x64）；架构预留 macOS（Intel + Apple Silicon） |
| **最低系统** | Windows 10 1809+（WebView2 运行时依赖） |

---

## 4. 范围边界（不做）

- ❌ Markdown 编辑功能（纯阅读器）
- ❌ 文件树浏览 / 文件名搜索
- ❌ 应用内自动更新（初期不做）
- ❌ 多窗口（单窗口多标签即可）
- ❌ 插件系统（初期不做）
- ❌ 云同步 / 账号体系

---

## 5. 验收标准

- [ ] 能通过 4 种方式（右键/拖拽/对话框/最近文件）打开任意 `.md` 文件并正确渲染
- [ ] TOC 目录正确生成，点击跳转、滚动联动正常
- [ ] 深浅主题切换即时生效，重启后保持选择
- [ ] 相对路径图片、绝对路径图片、网络图片均能正确显示
- [ ] 图片点击 lightbox 放大，支持缩放与拖动
- [ ] 全文搜索能高亮匹配并提供上下导航
- [ ] 4 种导出方式（PDF/HTML/打印/复制）均正常工作
- [ ] 文件外部修改后自动刷新，无肉眼可见抖动
- [ ] 命令行参数传入文件路径能直接打开
- [ ] 安装包能正确注册右键菜单与文件关联
- [ ] 前端代码无 `platform` 硬编码，Mac 适配仅差后端命令实现
